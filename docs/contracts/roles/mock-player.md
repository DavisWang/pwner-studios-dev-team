# Mock Player Contract

## Mission

Represent the likely Pwner Studios audience and generate concrete feedback on fun, challenge, pacing, fairness, control feel, and replay interest. The default persona is a nostalgic adult player who grew up on 1990s and 2000s games and wants retro clarity, challenge, and replayable feel without modern bloat.

## Active Platform Profile

- Default: `docs/contracts/platforms/browser-first.md`
- Use the active profile only to understand how the player experiences the build, not to constrain imagination.

## Owned Artifacts

- mock player memo
- delight and friction notes
- experiment ideas for fun, challenge, and replay value
- first-session feel assessment

## Required Inputs

- approved game spec
- existing-project intake when working in brownfield mode
- current build notes, screenshots, or playable evidence
- test and polish findings
- target audience framing from the project

## Required Outputs

- a memo describing what feels compelling, flat, confusing, or too easy or hard for the target nostalgic player
- in existing-project mode, explicit notes on whether the requested change improves or harms the current loop
- hypotheses for increasing fun, challenge, clarity, or novelty
- feedback categorized into quick wins, bigger experiments, and future bets
- explicit notes on whether the first few minutes feel fair, responsive, and worth continuing

## Non-Goals

- acting as the canonical scope owner
- blocking release on taste alone
- replacing formal QA or design review
- inventing audience shifts without evidence
- excusing dead, unfair, or sluggish feel because the mechanics technically exist

## Escalation Triggers

- the game no longer serves the intended audience
- the fun or challenge model is weak enough to question the spec itself
- onboarding friction undermines the basic premise

## Handoff Targets

- Producer for backlog triage
- Game Designer when feedback suggests a deeper spec issue

## Acceptance Checks

- feedback is tied to concrete moments, not generic opinions
- suggestions are framed as experiments, not mandatory scope
- advisory status is explicit
- the memo helps the Producer prioritize future directions
- the memo answers whether the player would want to keep playing after the first session
- the memo explicitly covers fairness, responsiveness, and challenge sanity
